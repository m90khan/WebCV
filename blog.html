How Js Code Runs?

In a conceptual overview. To run a Js code, a Js engine is needed. The code is passed through a parser which reads and checks the code. The parser knows the Js rules and knows the validation. if there is some error it throws the error on console and stops the execution. if everything is fine, its build a data structure in the form of parse tree mainly abstract syntax tree.

From here, it's no longer a Js code or to say it is now a set of instructions because the code is transalated into machine readable and now the machine executes the code and you get to see the functionality on the screen.

However, behind the scens there are execution contect and execution stack. Execution context is a container so to say in which a part of the code is validated and executed and stores variables if any. and By default, it will be global execution context which executes all code that is not inside any function. you can also call it global object which in here will be window object. the code that is inside the function get its execution context. which gets placed on top of global Execution context which forms execution stack.

as mentioned before, you can associate execution context as an global object which has 3 properties

  1. Variable object(VO) - Function arguments, (Function Declaration and Variable Declaration) known as Hoisting

  2. Scope Chain - it answers that where we can access a certain variable? . there is also lexical scoping. currrent to parent access.

  3. Scope Chain - it answers that where we can access a certain variable? . there is also lexical scoping. currrent and parent access.

When a function is called, a new execution context is placed in execution stack which happened in two phases . Creation and execution phase. as obvious, in the creation phase all the 3 properties are created and in the execution phase, the code of the current Execution context is run line by line.
